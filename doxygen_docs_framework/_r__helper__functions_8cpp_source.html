10 /*
11  * !!!!! IMPORTANT NOTE !!!!!:
12  * This is a C++ interface to the C library in R. C++ supports exceptions, but C does not.
13  * R's C library requires you to explicitly PROTECT() and UNPROTECT() memory for garbage collection.
14  * If care is not taken, and an exception is thrown after memory has been PROTECTed, but before it has been UNPROTECTed, UNPROTECT will not be called.
15  *
16  * Every effort should be made to not use exception-throwing functions between PROTECT and UNPROTECT pairs.
17  * If an exception-throwing function is used between pairs, a means of ensuring that UNPROTECT is called when exceptions are thrown is required.
18  *
19  * The ScopeGuard pattern could be used which would store protect_count internally, and call UNPROTECT(protect_count) in the destructor.
20  * It would be used as follows:
21  *
22  * guard = ProtectGuard(); // ProtectGuard is an implementation of the ScopeGuard pattern.
23  * PROTECT(something);
24  * guard.increment_protect_count(1); // Call this immediately after each set of PROTECT calls using the appropriate number of items to protect.
25  *
26  * The ProtectGuard destructor will be called when the function exists, regardless whether exceptions were thrown.
27  *
28  * Reference:
29  * For std::unordered_map and std::vector, begin(), end(), size() and operator[() are guaranteed to not throw exceptions.
30  *
31  */
