std::string  module_name

Checks over an error_map for problems.

The error_map should pair a string description of an important criterion which must be met, e.g. "Parameter_A must be non-zero", with a boolean indicating whether it has been met. If the condition was not met, an appropriate error message will be generated indicating both the problem and the module in which it occurred.

Adds class name prefixes to each quantity name indicating that it was calculated for a certain class.

E.g., a multilayer canopy model may calculate different incident light irradiance values for sunlit and shaded leaves. In this case "sunlit" and "shaded" would be the two leaf classes.
